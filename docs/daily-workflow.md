# Daily Workflow

This file is the placeholder for step 5 in the implementation plan.

Current minimum flow:
1. Open or create `sessions/{yyyy-mm-dd}.yaml`
2. Start a new problem with `problem-intake` or resume an existing one
3. Solve inside `problems/{platform}/{problem_slug}/workspace/`
4. Save an attempt snapshot with `attempt-tracker`
5. Collect editorial and ranker notes into `references/`
6. Generate grounded review with `evidence-review`
7. Update the session queue and next actions with `daily-manager`
