
## 문제 파악

1<= N <= 1000
1<= K <= 1e9

## 구해야 되는 값 정리하기

X_i는 0에서 K까지의 값을 갖게 되고, sum(X_i)  = K 다.

B_i = A_i + X_i라고 하자.

f(X) = prod (B_i) % 998244353

전체 가능한 경우의 수
(N+K-1)C (N-1) (C는 combination)

## 접근 방향

### Hint 얻기전의 접근 방식

f(X)의 기대값을 구하는게 최종적인 목표다.
기대값을 구하는 방법은 여러가지가 존재할 것 같다.

1. 말 그대로 모든 경우의 수를 모두 탐색해서 구하는 방식. 단, 해당 방식은 너무 큰 복잡도를 갖는 방식이 된다.
2. B_i가 기대값에 미치는 영향도를 파악하는 방식을 탐색한다. 하지만, B_i가 f(X)에 미치는 영향을 단순히 일반적으로 구하기 어려울 수 있다. B_i의 값이 정해졌다고 해보자. 이때, i를 제외하고 나머지 B의 집합을 B\`이라고하자. f(B\`)을 다시 구해야 되는 문제가 발생한다. 그리고 X_i의 값에 따라서 K도 변경되므로.. 단순 Sub문제라고 정의하기는 꽤 어려운 상황이다.


### Hint 접근 이후 접근 방식

prod(B_i)가 아닌, prod(A_i + X_i)로 문제를 바라본다.
위와 같이 문제를 바라보게 되면, A_i,X_i가 동시에 곱이 되는 경우는 없다는걸 알 수 있다.
X가 s번 선택되면, A는 N-s번 선택되는 걸 알 수 있다. 따라서, 다음과 같은 확률 기댓값으로 문제를 바라볼 수 있다.

X가 s번 선택되었다고 해보자. 이때, X들의 합은 0부터 K까지이고, A들의 곱의 합은 O(N^2) dp를 통해서 구할 수 있는 값이 된다.
해당 dp의 값의 합과 X의 prod 합으로 문제를 다시 바라볼 수 있게 된다. 그리고 X의 index는 고려하지 않고 s개가 있다는 사실만 중요하다.

s개의 확률변수가 존재하고, s개 확률 변수의 합은 M이고, 개별 확률변수의 값은 0 이상이라고 하자.
이때, g(M,s)를 구할 수 있으면 문제해결이 가능하다.
또는, 다음과 같이 문제를 해석할 수도 있을 것 같다. g(s)를 구하되, sum(X) <= K.
